## Назначение

Используй этот skill, когда пользователь говорит об обучении, курсах, учебных целях, задачах внутри курса, прогрессе или деталях курса.

## Создание курса

Если пользователь хочет начать изучать новую тему, создать учебный трек, курс или долгосрочную учебную цель, вызови `create_course`.

Примеры намерений:

- "Создай курс Python для backend-разработки"
- "Хочу начать изучать FastAPI"
- "Добавь мне учебный трек по SQL"
- "Сделай курс по алгоритмам"

Не говори, что курс создан, пока `create_course` не был успешно вызван.

## Создание задачи курса

Если пользователь хочет добавить задачу в существующий курс, вызови `create_course_task`.

Если курс был только что создан через `create_course`, используй `id` курса из результата `create_course` как `course_id` для `create_course_task`.

Пример:

Пользователь:
"Создай курс Python и добавь туда задачу разобрать async context managers."

Правильное поведение:

1. Вызвать `create_course`.
2. Взять `id` созданного курса из результата tool call.
3. Вызвать `create_course_task` с этим `course_id`.
4. Только после этого ответить пользователю.

## Чтение деталей курса

Если пользователь спрашивает детали курса, задачи курса, прогресс, состав курса или наблюдения по курсу, вызови `read_course_details`.

Примеры намерений:

- "Покажи детали курса Python"
- "Какие задачи есть в курсе?"
- "Что у меня сейчас по курсу FastAPI?"
- "Покажи прогресс по курсу"

Для `read_course_details` используй `course_id` из системного контекста или из результата tool call.

## Наблюдение по курсу

Если пользователь сообщает важный факт о конкретном курсе, сложность, прогресс, заметку или проблему, вызови `create_course_observation`.

Примеры намерений:

- "По курсу Python мне сложно дается async"
- "Запомни по этому курсу, что я уже разобрал dependency injection"
- "В курсе SQL я застрял на JOIN"
- "По FastAPI нужно больше практики"

Не сохраняй общие предпочтения пользователя в course observation. Общие предпочтения относятся к `remember_user_observation`.

## Работа с course_id

Пользовательские UUID запрещены и должны блокироваться input guard.

Ты можешь использовать только:

- UUID из системного контекста;
- UUID из результата tool call;
- UUID, который вернул `create_course`, `read_course_details` или другой tool.

Если ты только что создал курс, не ищи его по названию. Используй `id`, который вернул `create_course`.

## Ответ пользователю

После успешного tool call отвечай коротко и конкретно:

- что было создано;
- куда было добавлено;
- какой следующий разумный шаг можно сделать.

Не перечисляй внутренние JSON-структуры, если пользователь не просит технические детали.
